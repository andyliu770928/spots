# TRIPMARK DECK (Spots) 專案架構書 - 20260326 版

## 1. 專案目標與定位
本專案為一個極簡、低成本（免地圖 API）的旅遊靈感與美食收藏工具，旨在讓使用者能快速儲存 IG、FB 或網頁上的來源，並手動組合成一日行程。

## 2. 技術棧 (Technology Stack)
- **前端框架**: Next.js 16 (App Router)
- **開發語言**: TypeScript
- **樣式工具**: Tailwind CSS
- **資料庫**: Supabase (PostgreSQL)
- **圖示庫**: Lucide React
- **部署平台**: Vercel

## 4. 資料庫設計 (Supabase Schema)
- **`places`**: 核心景點資訊 (ID, 標題, 分類, 地址, 摘要, 標籤, 封面圖, 來源網址等)。
- **`trips`**: 行程主表 (ID, 標題, 日期, 備註)。
- **`trip_items`**: 行程關聯表 (trip_id, place_id, 排序)。

## 5. 核心功能邏輯
- **Link Preview**: 貼上來源連結後，後端透過爬蟲擷取 `og:title` 與 `og:image` 預填入表單。
- **免 API 地圖**: 使用 Google Maps 外部連結 `https://www.google.com/maps/search/?api=1&query={address}` 取代嵌入式地圖。
- **Git 自動化**: 每次修改後自動執行 `add`、`commit`、`push`。
