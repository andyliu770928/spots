# Spots 專案開發規則與記憶檔

## 核心規則
1. **自動化 Git 操作**：在修改或新增任何程式碼後，必須自動執行 `git add`、`git commit` 與 `git push`（確保 Vercel 自動部署同步）。
2. **進 Git 的詳細流程**：遵循標準作業程序，確保變更紀錄清晰。
3. **架構同步更新**：每當專案架構有任何修改（目錄結構、技術棧、資料表等），必須自動產生一份新的架構書（檔名格式：架構_YYYYMMDD.md）。


## Git 詳細流程
當發生任何程式碼變動時，應遵循以下標準作業程序：

1. **檢查狀態**：確認變動的檔案。
   - `git status`
2. **暫存變更**：
   - `git add .`
3. **提交變更**：
   - `git commit -m "Auto-update: [具體修改的內容摘要]"`
4. **推送到雲端**：確保 GitHub 上的程式碼為最新，觸發 Vercel 自動部署。
   - `git push origin main`
